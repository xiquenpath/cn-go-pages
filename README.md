# cn-go-pages

## 项目简介

`cn-go-pages` 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何单个域名或具体网站，而是作为一个通用的页面存放和版本管理空间，方便对零散的前端页面进行集中维护与分发。

## 目录说明

```
.
├── pages/           # 存放各独立 HTML 页面文件
│   └── ...
├── assets/          # 页面引用的公共资源（CSS、JS、图片等）
│   └── ...
├── README.md        # 本文件
└── .gitignore       # Git 忽略规则
```

- **pages/**：每个子目录或文件对应一个独立页面，命名清晰，便于查找。
- **assets/**：存放页面间共用的样式、脚本或静态资源，避免重复。

## 页面归档说明

- 所有 HTML 页面均为独立、自包含或引用 `assets/` 下资源的静态文件。
- 页面内容不涉及特定商业、营销或敏感信息，仅用于技术展示、存档或学习参考。
- 页面可能包含基础的前端交互（如 JavaScript 处理），但无后端依赖。

## 维护说明

- 提交新页面时，请在 `pages/` 下创建对应目录或文件，并在必要时更新 `assets/` 中的公共资源。
- 建议为每个页面添加简短的注释或使用说明，方便其他贡献者理解。
- 请保持目录结构清晰，避免冗余文件。

## 贡献

欢迎提交 Issue 或 Pull Request 来改进页面内容、修复错误或添加新页面。请确保提交的内容符合本仓库的通用归档原则。
